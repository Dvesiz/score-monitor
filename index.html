01 How it works

从登录到提醒，
全部自动完成。

它不是另一个成绩查询页面，而是一个安静运行的监控器。配置一次账号与学期，之后只需要等待通知。

01

自动登录

调用本地 OCR 识别图形验证码，登录失败时自动刷新验证码并重试。

02

查询成绩

进入指定学年与学期的成绩页面，同时解析总评、平时分与卷面分。

03

比较变化

与本地缓存进行比对，只在出现新成绩或已有成绩发生变化时触发通知。

04

推送手机

通过 Bark 将课程名和成绩发送至 iOS 通知中心，程序继续进行下一轮监控。

02 Real output

看到真实结果，
而不是概念图。

终端记录每一步查询过程；当成绩变化时，完整结果会直接出现在 Bark 历史消息里。

03 Built for focus

少盯一次页面，
多做一点正事。

所有能力都围绕一件事设计：准确发现成绩变化，然后尽快告诉你。

本地验证码识别

使用 ddddocr 在本机识别验证码，不调用外部识别 API。

平时分与卷面分

从 VIEWSTATE 中解析 PSCJ 与 QMCJ，并与课程总成绩合并展示。

只通知真正的变化

保存上一次成绩快照，避免每轮查询都发送重复消息。

持续监控

默认每 5 分钟检查一次，也可通过命令行临时调整间隔。

临时切换账号

支持通过命令行参数切换学生账号，不覆盖已保存的默认配置。

复用现有浏览器

优先使用系统中的 Chrome 或 Edge，无需额外下载完整浏览器。

04 Quick setup

四步完成配置。

Windows 10 / 11 可直接运行打包版本。Bark 仅用于 iPhone 推送，不填写也能在程序中查看成绩。

1

连接校园网络

校内连接校园网；校外先登录学校 VPN，并在监控期间保持连接。

2

下载程序

从 GitHub Releases 下载 default.exe，放到任意目录。

3

填写配置

首次运行时输入学号、密码、学年、学期，以及可选的 Bark Key。

4

保持运行

程序建立首次快照后进入循环监控，发现变化时自动推送。

首次密码必须输入正确。连续登录失败可能导致教务账号被临时锁定，建议先在浏览器中确认密码。

05 Command line

需要时，
再多控制一点。

直接双击即可使用。下面的参数用于单次查询、重新配置或调试。

default.exe读取配置，按 5 分钟间隔持续监控
--once只检查一次，然后退出
--setup重新配置账号、学期与 Bark Key
--interval 10临时将检查间隔改为 10 分钟
--visible显示浏览器窗口，用于排查登录问题
--student-id ...临时切换账号，不修改默认配置
